Documentation

Install, configure, check, filter, and export.

This guide covers the day-one workflow for ProxyNova users and the build path for developers.

Install GeoLite2 Checking Scraping Exports Development

Install

Download the latest Windows executable from GitHub Releases and run it locally. ProxyNova does not require a hosted account or external dashboard.

Download:
https://github.com/savxzthc/ProxyNova/releases/latest/download/ProxyNova.exe

GeoLite2 setup

Offline country and city lookup requires MaxMind's GeoLite2-City database. Register for a free MaxMind account, download GeoLite2-City.mmdb, and place it at files/GeoLite2-City.mmdb before building from source.

Release builds should already include the required database if they were packaged by the project maintainer.

Checking workflow

  1. Paste one proxy per line in IP:PORT format or import a saved list.
  2. Select the protocols you want to test: HTTP, HTTPS, SOCKS4, SOCKS5.
  3. Set concurrency and timeout values that fit your network.
  4. Start checking and watch status, latency, anonymity, and geography populate in the results table.

Transparent proxies can still work, but they leak client identity through headers or origin detection. Treat anonymity grading as a filtering signal, not a security guarantee.

Scraping workflow

The scraper aggregates configured public source lists, deduplicates entries, and queues unique proxies for checking. Disable sources that are stale, unreachable, or outside your authorized use case.

Exports

Exports respect the current filter state. Use TXT for simple IP:PORT lists, CSV for spreadsheet workflows, and JSON for automation or downstream tooling.

Examples
  working-http-us.txt
  elite-low-latency.csv
  checked-results.json

Development mode

Use Wails dev mode when working on the frontend or Go backend.

git clone https://github.com/savxzthc/ProxyNova.git
cd ProxyNova
wails dev

For production builds, run wails build. The Windows executable is produced under build/bin.
