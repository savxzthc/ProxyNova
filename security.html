Responsible use

Proxy checking is powerful. Keep it authorized.

ProxyNova is intended for developers, researchers, security teams, and network administrators validating lists they are allowed to use.

Appropriate use

  • Cleaning proxy lists you own or are authorized to test.
  • Measuring latency, health, and anonymity for internal tooling.
  • Researching proxy behavior in a lawful lab or approved environment.
  • Auditing whether a proxy leaks headers or origin information.

Do not use it for

  • Spam, fraud, account abuse, or credential attacks.
  • Bypassing access controls or rate limits without permission.
  • Scraping or automation that violates a service's rules.
  • Masking traffic for harassment, exploitation, or evasion.

Security model

ProxyNova is not an anonymity product. It tests proxies and labels their observed behavior. A working or Elite result does not guarantee privacy, integrity, clean ownership, or safety of the proxy endpoint.

Operational guidance

Treat unknown public proxies as untrusted infrastructure. Do not send secrets, credentials, private traffic, production tokens, or sensitive browsing through random proxy endpoints.

Defensive methodology

For internal validation, keep source lists documented, cap concurrency to avoid accidental service impact, monitor outbound traffic, and preserve logs that explain what was tested and why.

Vulnerability reporting

Report code or packaging vulnerabilities through the GitHub repository so maintainers can triage them with the relevant version, platform, reproduction steps, and impact.
